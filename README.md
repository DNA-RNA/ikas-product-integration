# 🔄 Multi-Store Product Synchronization Platform

> **Status:** 🚧 Currently under active development

A middleware platform built with **.NET 8** that automatically synchronizes products from a master İkas store to multiple independent İkas stores using XML feeds and the İkas Admin GraphQL API.

The platform periodically imports products from a master store, applies category-based filtering and pricing rules, then distributes products to multiple destination stores while maintaining synchronization logs and transfer history.

---

# 📌 Project Overview

The project is designed to automate product distribution across multiple e-commerce stores.

Instead of manually managing products on each store, a single **master store** acts as the source of truth.

The synchronization process consists of:

- Reading the XML feed generated by the master store
- Parsing and storing products
- Filtering products by category
- Applying store-specific pricing rules
- Mapping products to the İkas Product model
- Synchronizing products through the İkas Admin GraphQL API
- Logging all transfer operations

# 🔄 Synchronization Workflow

### XML Import Job

Runs periodically using Hangfire.

- Downloads XML feeds
- Parses products
- Performs SKU-based upsert
- Updates the local product database

---

### Product Transfer Job

For each destination store:

- Loads store configuration
- Filters products
- Applies pricing rules
- Maps products to İkas models
- Creates or updates products using GraphQL
- Stores synchronization logs

# 🔄 Çoklu Mağaza Ürün Senkronizasyon Platformu

> **Durum:** 🚧 Aktif olarak geliştirilmektedir.

Bu proje, **.NET 8** ile geliştirilen ve bir ana İkas mağazasındaki ürünleri XML veri kaynağı üzerinden okuyarak, belirlenen kurallara göre birden fazla İkas mağazasına otomatik olarak senkronize eden bir middleware (ara katman) uygulamasıdır.

Sistem; ürünleri XML üzerinden içe aktarır, kategorilere göre filtreler, mağazaya özel fiyatlandırma kurallarını uygular ve İkas Admin GraphQL API aracılığıyla hedef mağazalara otomatik olarak aktarır.

---

# 📌 Proje Hakkında

Bu projenin amacı, birden fazla e-ticaret mağazasındaki ürün yönetimini tek bir merkezden otomatik hale getirmektir.

Ana mağaza (**Master Store**) tüm ürünlerin yönetildiği kaynak sistem olarak çalışırken, diğer mağazalar yalnızca kendilerine ait kategorilerdeki ürünleri otomatik olarak alır.

Senkronizasyon süreci şu adımlardan oluşmaktadır:

- XML veri kaynağının okunması
- Ürünlerin parse edilmesi
- Ürünlerin veritabanına kaydedilmesi
- Kategori bazlı filtreleme
- Mağazaya özel fiyat marjlarının uygulanması
- Ürünlerin İkas veri modeline dönüştürülmesi
- İkas Admin GraphQL API üzerinden oluşturulması veya güncellenmesi
- Tüm transfer işlemlerinin kayıt altına alınması
